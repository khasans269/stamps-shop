# Подключение оплаты (ЮKassa) — этап 1

Оплата товара и доставки одним платежом через ЮKassa. Доставка на этом
этапе — фиксированная сумма (`DELIVERY_FLAT_FEE`).

## Как это работает

1. Покупатель заполняет форму на `/checkout` и жмёт «Перейти к оплате».
2. `POST /api/payment/create`:
   - пересчитывает сумму на сервере = товары + `DELIVERY_FLAT_FEE`;
   - записывает предзаказ в Google Sheets со статусом **«ожидает оплаты»**;
   - создаёт платёж в ЮKassa и возвращает ссылку на оплату.
3. Покупатель уходит на страницу оплаты ЮKassa, платит, возвращается на
   `/checkout/success`.
4. ЮKassa шлёт уведомление на `POST /api/payment/webhook`. Роут
   **перепроверяет статус обратным запросом** к API ЮKassa (телу уведомления
   не доверяем) и на `payment.succeeded`:
   - шлёт продавцу в Telegram «Заказ оплачён»;
   - меняет статус заказа в Google Sheets на **«оплачено»**.

Источник истины — webhook, а не редирект. Даже если покупатель закроет
вкладку, оплаченный заказ всё равно подтвердится через webhook. А если
webhook на время упадёт — ЮKassa повторяет его ~сутки, плюс предзапись
«ожидает оплаты» с `payment_id` остаётся в таблице для ручной сверки.

## Переменные окружения (Vercel → Project Settings → Environment Variables)

| Переменная | Что это |
|---|---|
| `YOOKASSA_SHOP_ID` | shopId из ЛК ЮKassa |
| `YOOKASSA_SECRET_KEY` | секретный ключ из ЛК ЮKassa |
| `DELIVERY_FLAT_FEE` | фикс доставки, ₽ (целое число) |
| `YOOKASSA_SEND_RECEIPT` | `true` — передавать чек в ЮKassa; иначе не передавать |
| `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` | уже настроены |
| `SHEETS_WEBHOOK_URL` | уже настроен |

Ключи в код/git не попадают. См. `.env.example`.

## Регистрация webhook в ЮKassa

В личном кабинете ЮKassa (или через API) укажите URL для HTTP-уведомлений:

```
https://<ваш-домен>/api/payment/webhook
```

Подпишитесь на события: `payment.succeeded`, `payment.canceled`,
`refund.succeeded`.

## Тестовый режим (рекомендуется сначала)

У ЮKassa есть тестовые `shopId`/секрет — тот же API, но деньги не
списываются. Порядок: выставить тестовые ключи в Preview-окружении Vercel,
пройти оплату тестовой картой, убедиться, что webhook доходит и статус в
Sheets меняется. Затем поменять на боевые ключи.

## Чек для самозанятого (НПД)

Пока автосервис чеков в ЮKassa **не подключён**, держите
`YOOKASSA_SEND_RECEIPT` пустым — чек пробивается вручную в приложении
«Мой налог» на полную сумму (товар + доставка).

Когда подключите автосервис чеков для самозанятых:

1. Поставьте `YOOKASSA_SEND_RECEIPT=true`.
2. **Сверьте формат чека** в `lib/yookassa.ts` → `buildReceipt`: значения
   `vat_code`, `payment_subject`, `payment_mode` для схемы самозанятого могут
   отличаться от заданных по умолчанию (сейчас — «без НДС», товар/услуга,
   полный расчёт). Точные значения — в доках/ЛК ЮKassa.

Чек всегда формируется на **полную сумму** (товар + доставка вместе),
доставка отдельным тарифом не выделяется.

## Что нужно доработать в Google Apps Script

Роут создания платежа и webhook шлют в `SHEETS_WEBHOOK_URL` те же данные,
что и раньше, плюс служебные поля:

- `action` — `"create"` (новая строка) или `"update"` (обновить строку с тем
  же `orderId`);
- `status` — `"ожидает оплаты"` | `"оплачено"` | `"отменён"` | `"возврат"`;
- `paymentId` — id платежа в ЮKassa.

Чтобы статус реально обновлялся (а не добавлялась вторая строка), в Apps
Script нужно:

1. Добавить в таблицу колонки **Статус** и **Payment ID**.
2. В `doPost` смотреть `data.action`:
   - `"create"` — как сейчас, добавить строку (записав `status` и `paymentId`);
   - `"update"` — найти строку по `orderId` и обновить в ней `status`/`paymentId`.

Если это пока не сделано — ничего не сломается: `update` просто добавит
отдельную строку, данные не потеряются, но статусы будут в разных строках.

## Известные упрощения этапа 1

- Полный состав оплаченного заказа продавец смотрит в строке Google Sheets
  (в Telegram после оплаты приходит короткое уведомление: кто, телефон,
  сумма, `payment_id`). Это сделано намеренно — в metadata платежа весь
  заказ не помещается, а детали и так есть в таблице.
- Страница `/checkout/success` очищает корзину при показе. Если оплата была
  отменена, корзину можно собрать заново.
- Точный расчёт доставки через API Ozon/Яндекс — это этап 2 (см. ТЗ).
