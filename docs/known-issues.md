## Кеш оптимизированных картинок (next/image)

**Симптом.** При смене файла под тем же путём (или после переноса файла в
подпапку) на странице ещё какое-то время показывается старая картинка.
Пример: в коде указан путь `/images/individual/st05.jpg`, файла по этому пути
уже нет (он переехал в `stamp_example/`), а картинка всё равно грузится — она
отдаётся из кеша оптимизатора `next/image` (кеш по URL) и/или из кеша браузера.

**Влияние.** Мешает только на этапе правок. На боевом сайте после чистой
сборки (деплоя) всё корректно.

**Как обойти сейчас.**
- Локально: остановить dev → `rm -rf .next` → `npm run dev`.
- В браузере: жёсткое обновление (Cmd+Shift+R) или режим инкогнито.

**Возможный фикс на будущее.** Пересмотреть кеш-заголовки для `/_next/image`
или revalidate; при необходимости — вынести чтение папок в динамический
рендер. Пока не критично.
