# stamps-shop — интернет-магазин штампов для керамики

## О проекте
Сайт для прямых продаж керамических штампов, текстурных роликов и инструментов.
Владелец: Аскар (самозанятый). Раньше продавал на Ozon, теперь переходит на собственный сайт, чтобы не платить комиссии и собирать аудиторию.
Целевая аудитория: керамисты и гончары — от любителей до небольших студий.

## Технологический стек
- Next.js 16 (App Router)
- React 19
- TypeScript
- Tailwind CSS v4
- Без `src/` — приложение живёт в `app/`
- Импорт-алиас: `@/*` → корень проекта

## Структура проекта

```
app/                  # Страницы (App Router)
  layout.tsx          # Общий layout (шрифты, meta, header/footer когда появятся)
  page.tsx            # Главная
  catalog/
    page.tsx          # Каталог — грид всех товаров
  product/[slug]/     # (будет) Карточка товара
  cart/               # (будет) Корзина
  checkout/           # (будет) Оформление заказа
  faq/                # (будет) FAQ и гайды
  legal/              # (будет) Оферта, политика конфиденциальности
components/           # React-компоненты (ProductCard, Header и т.д.)
data/
  products.json       # Каталог товаров — единый источник правды
lib/                  # (будет) Утилиты: форматирование цен, работа с корзиной
types/
  index.ts            # TypeScript типы (Product, Category)
public/
  images/products/    # Фото товаров — имя файла = slug товара
```

## Данные товаров
Товары хранятся в `data/products.json`. Чтобы добавить новый:
1. Открыть `data/products.json`.
2. Скопировать любой существующий объект из массива `products`.
3. Поменять поля: `id`, `slug`, `name`, `price`, `description`, `category`, `inStock`.
4. Если есть фото — положить его в `public/images/products/<slug>.jpg` и добавить поле `"image": "/images/products/<slug>.jpg"`.

**Правила для полей:**
- `id` — уникальная строка. Формат не принципиален, главное уникальность. Используем `p001`, `p002` для простоты.
- `slug` — часть URL. Только латиница в нижнем регистре, цифры и дефисы. Пример: `alfavit-kirillica-malyy`.
- `price` — число, в рублях, без копеек. `1800` = 1 800 ₽.
- `category` — одно из: `alphabets`, `patterns`, `rollers`, `tools`.
- `inStock` — `true` если есть в наличии, `false` если закончился.
- `image` — опционально. Если поле отсутствует, на карточке покажется placeholder.

## Категории

| ID категории | Название в UI |
|--------------|---------------|
| `alphabets`  | Алфавиты |
| `patterns`   | Штампы с узорами |
| `rollers`    | Текстурные ролики |
| `tools`      | Инструменты |

Чтобы добавить новую категорию: добавить значение в тип `Category` в `types/index.ts` и подпись в `CATEGORY_NAMES` там же.

## Правила для AI-ассистента

**Что нельзя без явного подтверждения:**
- Удалять или редактировать файлы в `app/legal/` — это юридические документы.
- Удалять товары из `data/products.json` (добавлять и редактировать — можно).
- Менять содержимое `data/products.json` напрямую, если пользователь не попросил. Предпочитать редактирование через пользователя.

**Что всегда делать:**
- Все тексты в UI — на русском. Цены — в рублях с символом ₽ (например: `1 800 ₽`, через неразрывный пробел или через `toLocaleString('ru-RU')`).
- Mobile-first вёрстка: писать Tailwind-классы начиная с мобильной версии, адаптив через `sm:`, `md:`, `lg:`.
- Серверные компоненты по умолчанию. `"use client"` — только там, где действительно нужна интерактивность или браузерные API.
- Стили — только Tailwind. Не добавлять новые `.css` файлы без явной просьбы.
- Никакого `localStorage` / `sessionStorage` на сервере. Для корзины — отдельный клиентский контекст (React Context + `useEffect`).
- После любого значимого изменения — предлагать `git commit` с понятным сообщением на русском.

**Качество кода:**
- Понятные имена переменных и функций.
- Комментарии на русском для неочевидных мест.
- Не переусложнять — пользователь новичок в вебе и должен иметь возможность читать код.

## Про пользователя (важно для AI)

Аскар — новичок в вебе. Первое приложение, которое он делает «всерьёз». Это значит:
- Не обижаться на базовые вопросы, отвечать без снисходительности.
- Объяснять незнакомые термины в контексте (домен, хостинг, SSL, миграция и т.д.).
- Избегать «умного» кода, который экономит две строчки, но делает код нечитаемым.
- Предлагать варианты с плюсами/минусами, а не давать один готовый ответ.
